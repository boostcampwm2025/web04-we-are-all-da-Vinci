# 앱인토스 퍼널 이벤트 표준화

관련 이슈: #393

## 목적

앱인토스 Analytics, GA4, 서버 로그에 이미 쌓이는 운영 데이터를 같은 퍼널 기준으로 읽기 위한 문서예요. 새 분석 체계를 크게 만드는 게 아니라, 현재 수집 중인 이벤트를 운영자가 이탈 지점 분석에 바로 활용할 수 있게 정리해요.

이 문서는 `client-toss`에서 발생시키는 퍼널 이벤트의 이름, 성공/실패 기준, 주요 파라미터를 팀이 함께 리뷰하는 기준으로 사용해요.

## 데이터 소스별 역할

| 데이터 소스 | 역할 | 확인할 내용 |
| --- | --- | --- |
| 앱인토스 Analytics | 토스 WebView 안의 화면 진입, 클릭, 노출 이벤트를 확인해요. | 플랫폼 콘솔에서 유입, 노출, 클릭 흐름을 교차 검증해요. |
| GA4 | 퍼널 전환율과 실험 전후 변화를 빠르게 봐요. | `trackClick`, `trackScreen`, `trackImpression`에서 fan-out된 이벤트를 기준으로 봐요. |
| 서버 로그/API | 실제 상태 변경 성공 여부를 확인해요. | 플레이 시작, 그림 등록, 광고/공유 보상처럼 서버 상태가 바뀌는 액션은 서버 성공 여부를 함께 봐요. |

## 운영자가 읽는 기준

- `attempt` 대비 `success`가 낮으면 API, 기회 차감, 광고/공유 보상, 네트워크 문제를 먼저 확인해요.
- 화면 진입 이벤트끼리 급격히 떨어지면 라우팅, 뒤로가기, 백그라운드 전환, 로딩 상태를 확인해요.
- 클릭은 많은데 성공이 낮으면 서버 실패, 인증 재발급 실패, SDK 실패처럼 완료 조건을 막는 문제를 확인해요.
- 화면은 많이 봤는데 클릭이 낮으면 CTA 문구, 보상 기대감, 랭킹/공유 동기를 확인해요.

## 기준 퍼널

| 단계 | 의미 | 클라이언트 이벤트 | 함께 볼 서버/API 기준 | 주요 이탈 해석 |
| --- | --- | --- | --- | --- |
| 1. 앱 진입 | 사용자가 앱에 들어와요. | `page_view`, `attribution_first_touch` | 사용자 조회, 로그인 요청 | 유입은 있는데 다음 단계가 낮으면 첫 화면, 로그인, 로딩 문제를 의심해요. |
| 2. 오늘 그림 시작 시도 | 무료 기회, 광고, 재도전으로 플레이 시작을 시도해요. | `play_start_attempt` | `startPlay` 요청 | 시도 자체가 낮으면 CTA, 기회 문구, 자동 시작 정책을 봐요. |
| 3. 오늘 그림 시작 성공 | 서버가 프롬프트를 반환하고 플레이가 시작돼요. | `play_start_success` | `startPlay` 성공 응답 | 성공률이 낮으면 기회 부족, 프롬프트 누락, 인증/네트워크 문제를 봐요. |
| 4. 외우기 화면 도달 | 제시 그림을 보는 화면에 도달해요. | `memorize_view` | 없음 | 시작 성공 대비 낮으면 라우팅이나 route state 전달 문제를 봐요. |
| 5. 그리기 화면 도달 | 캔버스 화면에 도달해요. | `drawing_view` | 없음 | 외우기 대비 낮으면 카운트다운 중 이탈, 뒤로가기, 백그라운드 전환을 봐요. |
| 6. 첫 획 발생 | 사용자가 실제로 그림을 그리기 시작해요. | `drawing_first_stroke` | stroke 저장 요청 여부 | 그리기 화면 대비 낮으면 캔버스 터치 반응성과 첫 행동 유도 문구를 봐요. |
| 7. 제출 전 결과 화면 도달 | 완성 그림과 점수 확인 화면에 도달해요. | `submitted_view` | 없음 | 첫 획 대비 낮으면 제한시간, 자동 제출, 스코어링 오류를 봐요. |
| 8. 등록 클릭 | 사용자가 그림 등록 의사를 표현해요. | `submitted_submit_click` | 그림 등록 요청 | 결과 화면 대비 낮으면 보상/랭킹 기대감과 등록 CTA 문구를 봐요. |
| 9. 등록 성공 | 서버가 그림 저장과 랭킹 반영을 완료해요. | `submitted_submit_success` | 그림 등록 성공 응답 | 클릭 대비 낮으면 API 실패, 인증 재발급, 유사도 계산 문제를 봐요. |
| 10. 랭킹 확인 | 등록 후 경쟁/비교 화면을 확인해요. | `ranking_view`, `ranking_item_click` | 랭킹/그림 상세 조회 | 등록 성공 대비 낮으면 홈 복귀 후 랭킹 유도와 탭 가시성을 봐요. |
| 11. 공유/재도전 | 재방문 또는 바이럴 루프로 들어가요. | `share_score_selected`, `share_invite_selected`, `play_start_success` with `source=submitted_replay` | 광고/공유 보상 성공 여부 | 랭킹 확인 대비 낮으면 자랑거리, 초대 보상, 광고 보상 UX를 봐요. |

## 이벤트 이름 기준

기존 GA4/앱인토스 대시보드와의 호환을 위해 기존 이벤트 이름은 유지해요. 새 이벤트를 추가할 때는 아래 접미사를 기준으로 맞춰요.

| 접미사 | 의미 | 예시 |
| --- | --- | --- |
| `_attempt` | 사용자가 액션을 시도했거나 자동 액션이 시작됐어요. | `play_start_attempt` |
| `_click` | 사용자가 버튼이나 항목을 눌렀어요. | `submitted_submit_click` |
| `_success` | 서버 상태 변경 또는 핵심 액션 완료가 확인됐어요. | `play_start_success`, `submitted_submit_success` |
| `_failed` | 액션 실패가 확인됐어요. | `play_start_failed`, `submitted_submit_failed` |
| `_view` | 특정 화면에 도달했어요. | `memorize_view`, `drawing_view` |
| `_impression` | 광고나 컴포넌트가 화면에 노출됐어요. | `banner_ad_impression` |

## 공통 파라미터 기준

| 파라미터 | 의미 | 사용 예 |
| --- | --- | --- |
| `source` | 액션 진입 경로예요. | `auto`, `cta`, `retry`, `submitted_replay` |
| `prompt_id` | 오늘 프롬프트 식별자예요. | 시작 성공, 외우기/그리기 화면 |
| `reason` | 실패 사유예요. | 서버 에러 메시지 또는 클라이언트 분기 |
| `score` | 클라이언트가 본 점수예요. | 등록 클릭/성공/실패 |
| `stroke_count` | 제출 시 획 수예요. | 등록 클릭/성공/실패 |
| `promotion_granted` | 포인트 지급 여부예요. | 등록 성공 |
| `ad_group_id` | 광고 그룹 식별자예요. | 광고 보상 시도/성공/실패 |
| `chance_count` | 보상 후 남은 기회 수예요. | 광고/공유 보상 성공 |

## 리뷰할 때 볼 점

- 이벤트 이름이 `FUNNEL_EVENTS`에 정의되어 있는지 확인해요.
- 클릭 이벤트만 추가하고 실제 성공/실패 이벤트를 빠뜨리지 않았는지 확인해요.
- 운영자가 퍼널을 볼 때 필요한 `source`, `reason`, `prompt_id` 같은 파라미터가 있는지 확인해요.
- 앱인토스 Analytics, GA4, 서버 로그 중 어떤 데이터가 최종 성공 기준인지 헷갈리지 않는지 확인해요.
- 서버 로그의 실제 명칭이 이 문서의 설명과 다르면 서버 코드 기준에 맞춰 문서 표현을 조정해요.
